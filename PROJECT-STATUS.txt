LATEST UPDATE (May 14, 2026) - CSS Specificity Debugging & Fixes
================================================================================

✅ DEBUGGED & FIXED CSS SPECIFICITY CONFLICTS

Problems Identified:
- Blog article pages have their own narrow body styles (max-width: 800px) that were interfering with full-width nav and footer.
- Footer background rule had lower specificity than other styles or was being ignored.
- Nav CSS was fragmented and not robust enough in article pages.

Fixes Applied to all 6 blog articles:
1. Made footer CSS highly specific: `footer, .site-footer { background: #003d82 !important; ... }`
2. Forced footer to full width (`max-width: none; width: 100%`) so it doesn't get constrained by article body styles.
3. Added `!important` on critical footer properties for reliability.
4. Ensured nav has proper full-width rules.
5. Cleaned up rule ordering and duplication where possible.

Result:
- Footer now reliably shows dark blue background on all pages.
- Nav layout is consistent (logo left, links right).
- No more visual conflicts between article content styles and site chrome (nav/footer).

All pages across the site now have consistent, conflict-free header and footer styling.
